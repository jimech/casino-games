Casino Platform Master Implementation Guide
1. Overview
This project is a portfolio-grade multi-game casino platform with a single shared wallet, a
modern casino-style UI, AI-driven personalization, strong security, and an admin/audit console.
The goal is to demonstrate full-stack product engineering across wallet architecture, game
engines, realtime systems, AI features, and compliance-oriented UX. The platform should feel
real, polished, and modular, while remaining a simulated or virtual-currency product for portfolio
purposes.
2. Product Goals
The platform should support multiple casino games through one wallet and one transaction
ledger. It should include a polished lobby, game detail pages, game play screens, wallet pages,
bonuses, VIP progression, support, and admin tooling. It should also include AI features that
improve discovery, personalization, fraud/risk awareness, and player experience without
controlling game fairness or wallet math.
3. Non-Goals
This implementation is not intended to be a real-money gambling product. It should not depend
on licensed payment rails, regulated casino operators, or production custody of funds. Game
outcomes, wallet balance changes, and settlement rules must remain deterministic and
auditable, with AI used only for recommendations, risk scoring, content ordering, and support
automation.
4. System Architecture
Use a modular architecture with clear domain boundaries:
● Frontend: Next.js or React with TypeScript.
● Backend: NestJS with TypeScript.
● Database: PostgreSQL.
● Cache and pub/sub: Redis.

7. Game Engine Design
The game engine should be modular and support multiple round-based games with shared
settlement flow.
Slots
Implement reel state, paylines, payout table, bonus rounds, free spins, and spin resolution.
Provide visible RTP metadata, theme variants, and a clear state machine for idle, spinning,
resolving, and complete.
Blackjack
Implement deck handling, player actions, dealer logic, hand scoring, and round settlement.
Support hit, stand, double, and split. Game state should be deterministic and reconstructable
from stored round events.
Roulette
Implement European roulette with number, color, odd/even, and high/low bets. The wheel result
should resolve through the game engine and then settle through the wallet service.
Poker
Implement Texas Hold’em round flow with hand ranking, pot tracking, community cards, and
dealer or table simulation. Store round snapshots and settlement events separately.
Crash
Implement a multiplier curve, auto-cashout, manual cashout, and crash resolution. Track the
multiplier progress as a real-time game state, but never let AI interfere with the multiplier
outcome.
